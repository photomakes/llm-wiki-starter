# Метод Карпаты (LLM-wiki)

**Суть:** знание компилируется агентом в перелинкованную вику в момент поступления
материала, а не переоткрывается из сырых документов при каждом запросе.
**Теги:** #принцип #метод #вика
**Обновлено:** 2026-07-12

## Содержание

Обычный подход (RAG) — при каждом вопросе искать по сырым документам заново. Метод
Карпаты переворачивает это: как только приходит новый материал, агент **сразу**
осмысляет его и вплетает в готовую вики — обновляет существующие страницы, создаёт
новые, проставляет кросс-ссылки. При запросе агент отвечает уже по вике, а не по
сырью.

Слой сырых источников (`raw/`) при этом остаётся неизменяемым — он первоисточник
для проверки фактов. Ценность вики — не в объёме, а в **плотности связей**: одна
страница = одна концепция, страницы ссылаются друг на друга `[[так]]`.

Формула Карпаты: «Obsidian — IDE, LLM — программист, вики — кодовая база». Obsidian
здесь лишь просмотрщик (граф и кликабельные ссылки); он ничего не обрабатывает и с
агентом не интегрируется — оба просто работают с одной папкой markdown-файлов.

Первоисточник — гист Карпаты `llm-wiki.md` (апрель 2026):
https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f
