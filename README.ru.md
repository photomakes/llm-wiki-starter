# llm-wiki-starter

**🇬🇧 English version: [README.md](README.md)**

---

Готовый скелет системы долговременной памяти и знаний для ИИ-агента. Развивает
[метод LLM-wiki Карпаты](https://gist.github.com/karpathy/442a6bf555914893e9891c11519de94f)
— но как один слой из четырёх (вику), а не всю систему: знание в этом слое
компилируется в перелинкованную вику в момент поступления, а не переоткрывается
из сырых документов при каждом запросе.

## Проблема, из которой это выросло

ИИ-агент (Claude Code и подобные) стартует каждую сессию **с чистого листа** — он
не помнит прошлых разговоров. Наивное лечение — свалить всё в один файл — ломается
за неделю: файл раздувается, факты дублируются, копии расходятся, и агент начинает
работать по той версии, которая попалась первой.

Путь уже пройден за тебя. Это не «смотрите, как я устроил», а рабочий скелет:
**не одно хранилище, а несколько слоёв, у каждого своя роль.** Клонируй, оставь
слои под свою ступень, заполни своим содержимым.

## Четыре слоя

| Слой | Что хранит | Роль | Где в репо |
|---|---|---|---|
| **Правила поведения** | инструкции агенту | «конституция», грузится в КАЖДУЮ сессию | [`CLAUDE.md`](CLAUDE.md) |
| **Память проекта** | точечные факты и фидбеки, 1 факт = 1 файл | привязана к проекту, подгружается по релевантности | [`projects/demo-project/memory/`](projects/demo-project/memory/) |
| **База знаний (вика)** | осмысленное **переносимое** знание | перелинкованная «энциклопедия» | [`wiki/`](wiki/) |
| **Навыки / методички** | передаваемые методы-умения | оформленные приёмы для повторного применения | [`share/`](share/) |

Что репо показывает наглядно:
- память лежит **не** в папке проекта (см. [`projects/README.md`](projects/README.md));
- один факт = один файл + индекс `MEMORY.md`;
- кросс-ссылки `[[...]]` между страницами вики;
- бланки в [`templates/`](templates/) — «бери и заполняй под себя».

## Лестница масштаба — нужно ли это ТЕБЕ

Слой заводится **в ответ на боль**, а не заранее. Забежишь вперёд — будешь платить
за поддержку (уборка, синхронизация) впустую. Найди свою ступень и посмотри, чего
достаточно сейчас и по какому сигналу пора выше.

| Ступень | Ты здесь, если… | Чего достаточно | Сигнал «пора выше» |
|---|---|---|---|
| **0. Разовое** | разовые задачи | ничего, память сессии | возвращаешься к тому же и пересказываешь контекст заново |
| **1. Один малый проект** | редкие сессии | 1 файл заметок (проектный `CLAUDE.md`) | заметок много, ищешь «а где я это записал» |
| **2. Один долгий проект** | копятся решения и фидбеки | память проекта: 1 факт = 1 файл + индекс `MEMORY.md` | появился ВТОРОЙ проект, знание из первого пригодилось бы |
| **3. Несколько проектов** | знания пересекаются | глобальная вика + типизация (переносимое → вика, проектное → память) | «записал в память одного, а нужно везде»; дубли между проектами |
| **4. Крупный проект** | у одного проекта свой большой объём | локальная `__wiki` в git проекта | память распухла, «журнал под видом памяти» |
| **5. Передаёшь другим** | наработал повторяемые приёмы | слой навыков `share/` + обязательный бэкап | приём сработал 3-й раз; страшно потерять всё |

**Мораль:** ступень определяется не амбициями, а числом проектов и плотностью
пересечений между ними. Один проект — даже большой — почти не требует слоёв; они
окупаются, когда знания перетекают между проектами (ступень 3).

> **Кому НЕ нужно:** одному малому или разовому проекту хватит одного файла заметок.
> Не тащи все папки — это плата за поддержку без отдачи.

## Быстрый старт

1. **Клонируй репозиторий** и открой его в своём агенте (Claude Code и т.п.).
2. **Определи свою ступень** по таблице выше.
3. **Оставь нужные папки, остальные удали:**
   - ступень 1–2 → `projects/<slug>/memory/` (+ `CLAUDE.md` как слой правил появляется рано);
   - ступень 3 → добавь `wiki/`;
   - ступень 4 → заведи локальную `__wiki` в git самого проекта (в этот минимум-репо не входит — про неё словами в [share/storage-architecture.md](share/storage-architecture.md));
   - ступень 5 → добавь `share/` + настрой бэкап.
4. **Замени демо-содержимое своим.** Бланки — в [`templates/`](templates/).
5. **Прочитай справочник** [share/storage-architecture.md](share/storage-architecture.md) —
   там принципы, грабли и «как завести с нуля».

## Границы (честно)

Это **авторская сборка известных паттернов**, а не готовый стандарт с одним именем.
Кирпичи индустриальные (единый источник истины, разделение ответственности, база
знаний по методу Карпаты, слои памяти). Конкретика — имена файлов (`CLAUDE.md`,
`memory/`), кодирование пути проекта в slug — специфика Claude Code; принцип «слои
по ролям + единый источник истины» переносится на любой агент, детали — нет.

Основной содержательный материал — на русском. Перевод методичек `share/` на
английский — отдельная задача. Лицензия: [MIT](LICENSE).
