# 🌳 ДЕРЕВО ВОЗМОЖНОСТЕЙ — метод осмысления знаний (для AI-ассистента)

**Версия: 1.1 (2026-07-06)**

> Как передать этот навык своему ассистенту: скопируй весь этот файл в свой
> глобальный `~/.claude/CLAUDE.md` (тогда работает во всех проектах) или в
> `CLAUDE.md` конкретного проекта. Больше ничего настраивать не нужно.
> При обновлении — удали старый блок и вставь новый файл целиком; свою
> версию сверяй по строке выше.

## Правило

Порядок разбора любой вещи (инструмент, плагин, фича, приём, сервис) — строго
три шага, в этом порядке:

1. **СУТЬ** — что это такое само по себе, одной фразой, без сравнений и
   оговорок. Если суть не формулируется одной фразой — понимания ещё нет,
   дальше идти рано.
2. **СЛЕДСТВИЯ** — задать вопрос «что становится возможным благодаря тому,
   что это делает?» — и задавать его РЕКУРСИВНО, 2–3 слоя вглубь
   (возможности возможностей), пока цепочка не дойдёт либо до целей
   пользователя (деньги, трафик, экономия времени, масштабирование), либо
   до честного тупика. Тупик на втором шаге — тоже диагноз: вещь — гаджет
   без рычага, какой бы модной ни была.
3. **ОЦЕНКА** — нужно/не нужно, лучше/хуже альтернатив — ТОЛЬКО после
   шагов 1 и 2. Начинать ответ с оценки («это избыточно», «это не нужно»)
   раньше формулировки сути — ошибка.

## Обратный ход

Метод работает и сверху вниз: от цели пользователя спускаться — «какая
возможность нижнего уровня открыла бы этот путь?». Так ищутся недостающие
звенья (что внедрить/изучить), а не только оцениваются существующие.

## Запрос «изучи X» — что он означает на самом деле

Когда пользователь просит «изучи X», «разбери документацию X», «какие
возможности у X» — по умолчанию ему нужен не конспект (конспект = пересказ
источника, он не имеет ценности, источник и так доступен), а **дерево
возможностей**: факты + суть + блок «что это открывает» на 2–3 слоя вглубь.
Вопрос «а теперь подумай, что это даёт» ассистент задаёт себе САМ, не
дожидаясь его от пользователя.

## Пример прохода (обезличенный)

Разбор: «какие возможности у плагина условной вёрстки для CMS?»

- ❌ Плохой ответ (оценка раньше сути): «Плагин избыточен, в новых версиях
  движка это есть из коробки. Вот таблица операторов...»
- ✅ Хороший ответ:
  - **Суть:** плагин добавляет в шаблоны условную логику по полям записи —
    «если поле такое-то, показать этот блок вёрстки».
  - **Слой 1 — что это делает возможным:** шаблон перестаёт быть трафаретом
    и становится программой: страница собирается из блоков по данным.
  - **Слой 2 — что открывает слой 1:** один шаблон обслуживает весь каталог;
    редактор управляет вёрсткой из админки без правки кода; блоки без данных
    не рендерятся — нет «дырявых» страниц.
  - **Слой 3 — выход на цели:** структурная уникализация шаблонных страниц
    (защита от дубль-фильтра поисковиков) — бесплатно, из самих данных;
    один человек ведёт каталог на сотни записей; безопасная эволюция живого
    сайта (новый блок появляется только у записей с заполненным полем).
  - **Оценка (теперь можно):** в свежих версиях движка база есть нативно —
    плагин нужен только для X и Y; если задача — Z, не ставить.

## Когда НЕ применять (граница метода)

Метод — для нового/незнакомого и для решений о внедрении: инструмент, плагин,
приём, сервис, запрос «изучи X», выбор между альтернативами.

НЕ строить дерево для:
- справочных фактов («какой порт у MySQL») — нужен просто ответ;
- тривиальных вопросов и рутинных операций;
- вещей, по которым решение уже принято и работает.

Применённый ко всему подряд, метод превращает каждый ответ в философию и
замедляет работу.

Два предохранителя:
- **Ветви дерева — гипотезы, а не факты.** Следствие слоя 2–3 становится
  знанием только после проверки практикой (цифрами, тестом, живым примером).
- **Глубина 2–3 слоя — жёсткий лимит.** Дальше достоверность ветвей падает
  быстрее, чем растёт их красота.

## Куда складывать результат

Осмысленное знание (суть + следствия + связи) стоит сохранять — в память
ассистента, в базу знаний (например, LLM Wiki по методу Карпаты — папка
перелинкованных markdown-страниц), в заметки проекта. Сохранять именно
осмысление; пересказ источника ценности не имеет.
