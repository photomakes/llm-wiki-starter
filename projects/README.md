# projects/ — память проектов (и где она физически лежит)

**Важнейшая деталь, на которой все спотыкаются: память проекта хранится НЕ в папке
проекта.** Есть два разных места:

1. **Папка проекта на диске** — сам проект (код, файлы). Памяти агента тут нет;
   вложенная `.claude/` бывает пустой или хранит только настройки.
2. **Служебная папка агента в профиле пользователя** — вот тут живёт всё
   «агентское»: глобальные правила и подпапка `projects/` с отдельной папкой на
   каждый проект, а в ней `memory/` (файлы памяти) и логи сессий.

Для Claude Code это `~/.claude/projects/<slug>/memory/`. Связь с проектом — по
имени: путь проекта кодируется в имя служебной папки заменой разделителей
(`:` `\` `/`) на дефисы. Например путь `D:\projects\myproject` →
папка памяти `D--projects-myproject`.

**Почему так:** память и история сессий — приватные данные о работе агента; их
держат в профиле пользователя, чтобы не засорять репозиторий проекта и чтобы они
случайно не попали в git. Вику, наоборот, кладут В папку проекта — чтобы она
версионировалась вместе с ним.

---

Папка [`demo-project/memory/`](demo-project/memory/) в этом репо **имитирует** ту
самую служебную папку — чтобы показать формат, который у тебя будет лежать в
профиле агента, а не в коде проекта. Внутри:

- `MEMORY.md` — индекс: одна короткая строка на каждый факт;
- отдельные файлы `*.md` — один факт = один файл (бланк — в
  [`../templates/memory-file.template.md`](../templates/memory-file.template.md)).

Подробнее о слоях и граблях — в
[`../share/storage-architecture.md`](../share/storage-architecture.md).
